Client Request
"We need a notifications feature for our app."

Based only on that information, which of these would you build? Click the one that matches what you'd think to make.

Developer A
In-App Toast Alerts
Built small popup banners that appear briefly in the corner of the screen whenever an event occurs, then fade out automatically.
Developer B
In-App Notification Bell
Built a bell icon in the header that shows a badge count. Clicking it opens a dropdown of recent activity.
Developer C
Browser Push Notifications
Implemented push notifications that appear in the OS notification tray even when the app isn't open.
Developer D
Weekly Email Digest
Built an automated email that sends users a summary of their activity at the end of each week.
All four of these are valid "notifications features." Every developer built something reasonable. Every developer built something different. Without a user story, there was no way to know which one was right.
What a user story and acceptance criteria would have specified
"As a registered user, I want to see a notification when someone replies to my post so that I can respond without missing activity in the app."
  • Given I am logged in, when someone replies to my post, then a badge count appears on the notification bell within 5 seconds.
  • Given I click the notification bell, when the dropdown opens, then I see the replier's name and the first 100 characters of their reply.
  • Given I have unread notifications, when I click one, then it is marked as read and the badge count decreases by one.
The user story rules out email digests, push notifications, and settings toggles. The acceptance criteria specify the bell, the badge, the dropdown, and the timing. Developer B's build is now the only correct one, and any developer on the team would build the same thing.
