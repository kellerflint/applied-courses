Before writing any code, your team will produce wireframes for the full app. A wireframe is a low-fidelity sketch of a UI. No colors, no final styling, no polished graphics. The goal is to agree on what goes where and how the user moves through the app before you start implementing.

Be sure to note or otherwise document how the user moves between pages and what triggers each transition (clicking a video, submitting a job, etc.).

## What to include on each wireframe

- All interactive elements (buttons, inputs, links) labeled
- Approximate layout and sizing
- Notes on any dynamic behavior (e.g. "updates live as slider moves")
- Routes/URLs for each page

## Tools

Any of these are fine:

- Pen and paper (take clear photos)
- Figma, FigJam, or Excalidraw
- draw.io, Miro, Whimsical
- Anything else that produces a clear, shareable image

## Deliverable

Create a GitHub repository for your project and add a `wireframes/` folder containing all your wireframe images and the flow diagram. Include both team members' names in the README. Submit a link to the repository on Canvas. Each team submits once.
