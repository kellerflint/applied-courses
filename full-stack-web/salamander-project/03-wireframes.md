Before writing any code, your team will produce wireframes for the full app. A wireframe is a low-fidelity sketch of a UI. No colors, no final styling, no polished graphics. The goal is to agree on what goes where and how the user moves through the app before you start implementing.

Be sure to note or otherwise document how the user moves between pages and what triggers each transition (clicking a video, submitting a job, etc.).

## What to include on each wireframe

- All interactive elements (buttons, inputs, links) labeled
- Approximate layout and sizing
- Notes on any dynamic behavior (e.g. "updates live as slider moves")
- Routes/URLs for each page

## Tools

Any of these are fine:

- [wireframe.cc](https://wireframe.cc/)
- Pen and paper (take clear photos)
- Figma, FigJam, or Excalidraw
- draw.io, Miro, Whimsical
- Anything else that produces a clear, shareable image

## Color Scheme

Now is a good time to start thinking about color, even if the wireframes themselves are black and white. Having a palette picked early saves a lot of guesswork once you start building.

Aim for 3 to 5 colors: a background, a primary action color, a text color, and one or two accents. A few tools that can help:

**[Adobe Color Wheel](https://color.adobe.com/create/color-wheel)**
Pick a base color and explore complementary, analogous, or triadic combinations. Good for finding colors that work together without a lot of trial and error.

**[Coolors Image Picker](https://coolors.co/image-picker)**
Upload an image with colors you like and it extracts a palette from it. Useful if you already have a visual direction in mind.

**[Realtime Colors](https://www.realtimecolors.com/)**
Apply your palette to a live site mockup so you can see how the colors actually feel in context before committing to them.

If your wireframes are in color, use your chosen palette directly. If they are black and white (totally fine at this stage), add a note to your submission listing the hex codes you plan to use for different parts of the page.

## Deliverable

Create a GitHub repository for your project and add a `wireframes/` folder containing all your wireframe images and the flow diagram. Include both team members' names in the README. Submit a link to the repository on Canvas. Each team submits once.
